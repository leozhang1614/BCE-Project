## 🎯 核心要求

### 一、流程红线（必须遵守）

1. ⚠️ **所有开发任务必须走工作流** - 禁止私下交付
2. ⚠️ **必须通过审核才能上线** - 执矩有一票否决权
3. ⚠️ **审核不通过回退到开发** - 不是上一个节点，是开发节点
4. ⚠️ **必须重新走完整流程** - 重新开发 → 重新验收 → 重新审核

### 二、角色职责（必须明确）

| 角色 | 人员 | 职责 | 输出 |
|------|------|------|------|
| **开发者** | 任务执行人 | 功能实现、自测 | 代码 + 自测报告 |
| **项目负责人** | 任务创建人/指定人 | 验收、需求符合度 | 验收报告 |
| **审核人** | 执矩 | 代码合规、系统稳定 | 审核通过/驳回 |

### 三、时限要求（必须遵守）

| 节点 | 时限 | 超时处理 |
|------|------|---------|
| **开发** | 按任务 deadline | 系统自动提醒 |
| **验收** | 开发完成后 2 小时内 | 通知开发者 |
| **审核** | 验收通过后 1 小时内 | 通知项目负责人 |

## 📋 流程详解

### 阶段一：开发（开发者）

**工作内容：**
1. ✅ 理解需求
2. ✅ 编写代码
3. ✅ 单元测试
4. ✅ 自测通过

**交付物：**
- 源代码
- 自测报告
- 开发文档

**完成标准：**
- 功能完整
- 自测通过
- 无严重 Bug
- 文档齐全

**禁止行为：**
- ❌ 不写自测报告
- ❌ 不写文档
- ❌ 硬编码配置
- ❌ 影响系统稳定

---

### 阶段二：验收（项目负责人）

**工作内容：**
1. ✅ 功能验证
2. ✅ 需求符合度检查
3. ✅ 用户体验测试
4. ✅ 编写验收报告

**交付物：**
- 验收报告
- 问题清单（如有）

**完成标准：**
- 功能符合需求
- 用户体验良好
- 无严重 Bug

**禁止行为：**
- ❌ 不验收直接通过
- ❌ 忽略严重 Bug
- ❌ 不写验收报告

---

### 阶段三：审核（执矩）

**工作内容：**
1. ✅ 代码合规性审核
2. ✅ 系统稳定性评估
3. ✅ OpenClaw 功能影响评估
4. ✅ 安全风险评估

**审核标准：**

**代码合规：**
- ✅ 符合代码规范
- ✅ 无硬编码配置
- ✅ 错误处理完整

**系统稳定：**
- ✅ 不影响系统稳定性
- ✅ 无内存泄漏
- ✅ 性能良好

**OpenClaw 兼容：**
- ✅ 不影响核心功能
- ✅ 符合架构设计
- ✅ 遵循开发规范

**交付物：**
- 审核报告（通过/驳回）

**权限：**
- ✅ 一票否决权
- ✅ 可直接回退到开发节点

## 🔄 流转规则

### 正常流程
```
开发完成 → 提交验收 → 验收通过 → 提交审核 → 审核通过 → 任务完成
```

### 驳回流程（重点！）
```
审核不通过 → 回退到开发节点 → 重新开发 → 重新验收 → 重新审核
```

**关键点：**
1. ❌ **不是回退到验收节点**
2. ✅ **直接回退到开发节点**
3. ✅ **需要重新走完整流程**

## ⏰ 执行要求

### 一、任务创建

**创建人：** 项目负责人

**必填信息：**
- 任务标题
- 任务描述
- 开发者
- 截止时间
- 优先级

**创建方式：**
- BCE 系统 Web 界面
- API 调用

### 二、任务确认

**开发者收到任务后：**
1. ✅ 立即确认任务
2. ✅ 评估工作量
3. ✅ 确认截止时间
4. ✅ 开始执行

**时限要求：**
- P0 任务：30 分钟内确认
- P1 任务：2 小时内确认
- P2 任务：24 小时内确认

### 三、进度反馈

**开发者必须主动反馈：**

| 任务时长 | 反馈频率 | 反馈内容 |
|---------|---------|---------|
| < 1 小时 | 完成后反馈 | 完成结果 |
| 1-4 小时 | 每 1 小时反馈 | 进度百分比 |
| 4-24 小时 | 每 2 小时反馈 | 进度 + 问题 |
| > 24 小时 | 每天反馈 | 详细进度报告 |

### 四、提交验收

**开发者完成开发后：**
1. ✅ 编写自测报告
2. ✅ 整理交付物
3. ✅ 提交验收申请
4. ✅ 通知项目负责人

**提交内容：**
- 源代码链接
- 自测报告
- 开发文档

### 五、提交审核

**项目负责人验收通过后：**
1. ✅ 编写验收报告
2. ✅ 提交审核申请
3. ✅ 通知执矩

**提交内容：**
- 验收报告
- 交付物清单

### 六、任务完成

**审核通过后：**
1. ✅ 系统自动标记完成
2. ✅ 通知所有人
3. ✅ 交付物归档

## 🚨 违规处理

### 一级违规（警告）

**行为：**
- ❌ 超时未确认任务
- ❌ 超时未反馈进度
- ❌ 交付物不完整

**处理：**
- 系统自动提醒
- 记录违规次数
- 纳入绩效考核

### 二级违规（通报）

**行为：**
- ❌ 不走工作流私下交付
- ❌ 不验收直接通过
- ❌ 忽略严重 Bug

**处理：**
- 通报批评
- 重新走流程
- 纳入绩效考核

### 三级违规（严重）

**行为：**
- ❌ 影响系统稳定性
- ❌ 破坏 OpenClaw 核心功能
- ❌ 故意绕过审核

**处理：**
- 严重警告
- 重新培训
- 影响年终绩效
