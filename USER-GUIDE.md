## 📋 任务流转流程

```
创建 (pending) → 分配 (assigned) → 执行 (executing) → 验收 (reviewing) → 完成 (accepted)
```

---

## 🎯 各角色操作指南

### 天枢/磊哥（管理者）

#### 1. 创建任务
1. 访问 `http://192.168.31.187:3000/bce-tasks.html`
2. 点击 "➕ 创建任务"
3. 填写：
   - 任务标题（必填）
   - 任务描述
   - 创建人（选择自己）
   - 优先级（P0/P1/P2）
4. 点击 "创建任务"

#### 2. 分配任务
1. 在任务列表中找到刚创建的任务
2. 点击 "分配" 按钮
3. 输入负责人姓名（匠心/司库/执矩/磐石/灵犀）
4. 点击确认

#### 3. 验收任务
1. 在任务列表中找到状态为 "待验收" 的任务
2. 点击 "验收通过" 按钮
3. 输入验收意见（可选）
4. 点击确认

---

### 匠心/司库/执矩/磐石/灵犀（执行者）

#### 1. 查看分配给我的任务
1. 访问 `http://192.168.31.187:3000/bce-tasks.html`
2. 在任务列表中查找自己的名字
3. 或者使用筛选功能

#### 2. 开始执行任务
1. 找到状态为 "已分配" 的任务
2. 点击 "开始执行" 按钮
3. 任务状态变为 "执行中"

#### 3. 提交验收
1. 完成任务后，找到状态为 "执行中" 的任务
2. 点击 "提交验收" 按钮
3. 输入交付物（如：代码完成、测试通过）
4. 任务状态变为 "待验收"

#### 4. 添加评论
1. 点击任务的 "查看详情" 按钮
2. 在评论区输入内容
3. 点击 "发送"

## 💬 任务评论系统

**使用场景：**
- 任务执行过程中遇到问题
- 需要与任务创建人沟通
- 更新任务进度
- 请求协助

**操作步骤：**
1. 点击任务的 "查看详情"
2. 在底部评论框输入内容
3. 点击 "发送"
4. 所有相关人员都能看到评论

---

## 📂 子任务拆分

**使用场景：**
- 大任务需要分解为多个小任务
- 不同子任务由不同人负责
- 需要跟踪每个子任务的进度

**操作步骤：**
1. 点击任务的 "查看详情"
2. 在子任务区域点击 "添加子任务"
3. 填写：
   - 子任务标题
   - 子任务描述
   - 负责人
4. 点击确认

## 🔔 飞书通知

**系统会自动发送飞书通知的场景：**
- ✅ 任务分配时
- ✅ 任务开始执行时
- ✅ 任务提交验收时
- ✅ 任务验收通过时

**通知内容包含：**
- 任务标题
- 当前状态
- 操作人
- 备注信息
- 查看详情链接

## ⚠️ 常见问题

### Q1: 我找不到分配给我的任务
**A:** 
1. 确认你登录的是正确的账号
2. 检查任务列表的筛选条件
3. 刷新页面（点击 "🔄 刷新" 按钮）

### Q2: 任务状态无法更新
**A:**
1. 确认你有权限操作此任务
2. 检查任务当前状态是否允许转移
3. 刷新页面后重试

### Q3: 没有收到飞书通知
**A:**
1. 检查飞书是否正常登录
2. 确认飞书 Webhook 已配置
3. 联系磐石检查飞书配置

### Q4: 子任务/评论功能无法使用
**A:**
1. 确认访问的是 BCE 任务管理页面（3000 端口）
2. 清除浏览器缓存后重试
3. 联系匠心检查 API 状态
