## 🎯 系统概述

### 什么是 BCE？

**BCE（北斗协同引擎）** 是一款基于 OpenClaw 的智能任务管理和协同系统，帮助团队高效分配任务、跟踪进度、确保任务按时完成。

### v3.3 核心改进

| 功能 | v3.2 | v3.3 | 提升 |
|------|------|------|------|
| **驳回回退** | ❌ 不支持 | ✅ 支持 | 审核不通过自动回退 |
| **流转规则** | 单向流转 | ✅ 双向流转 | 统一规则引擎 |
| **升级通知** | ❌ 无 | ✅ 5 次 + 升级 | 自动通知天枢 + 磊哥 |
| **流程制度** | ❌ 无 | ✅ 完整制度 | 明确各节点要求 |

## 📋 核心功能模块

### 1️⃣ 任务创建

**功能说明：** 创建任务并分配给负责人

**创建方式：**
- OCC 面板创建（推荐）
- API 创建
- Control Center 同步

**必填字段：**
| 字段 | 类型 | 说明 |
|------|------|------|
| title | string | 任务标题 |
| creator | string | 创建人 |
| assignee | string | 负责人 |
| priority | P0/P1/P2 | 优先级 |
| dueDate | datetime | 截止时间 |

**API 示例：**
```bash
curl -X POST http://localhost:3000/api/bce/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "title": "技术方案设计",
    "description": "完成 BCE 系统技术方案",
    "creator": "天枢",
    "assignee": "匠心",
    "priority": "P0",
    "dueDate": "2026-03-22T18:00:00Z"
  }'
```

---

### 2️⃣ 任务通知（三通道）

**功能说明：** 任务创建/变更时自动通知相关人员

**通知通道：**
| 通道 | 说明 | 时效 | 可靠性 |
|------|------|------|--------|
| **主通道** | OpenClaw sessions_send | 实时 | ⭐⭐⭐ |
| **备用通道** | 飞书卡片@负责人 | 即时 | ⭐⭐⭐⭐ |
| **可靠通道** | 文件系统邮箱 | 离线拉取 | ⭐⭐⭐⭐⭐ |

**通知类型：**
- 任务创建通知
- 任务确认提醒
- 任务转交通知
- 任务驳回通知
- 任务完成通知
- 升级通知（≥5 次驳回）

---

### 3️⃣ 任务确认

**功能说明：** 负责人确认接收任务

**确认时效：**
| 优先级 | 确认时限 | 超时处理 |
|--------|---------|---------|
| **P0** | 30 分钟内 | 系统自动提醒 |
| **P1** | 2 小时内 | 系统自动提醒 |
| **P2** | 24 小时内 | 系统自动提醒 |
| **P3** | 72 小时内 | 系统自动提醒 |

**超时升级：**
- 5 分钟：第 2 次提醒
- 15 分钟：通知创建人
- 30 分钟：标记"超时未确认"，上报磊哥

**确认方式：**
- OCC 协作页面点击 [确认接收]
- API 确认
- 飞书回复"收到 + 姓名"

**API 示例：**
```bash
curl -X POST http://localhost:3000/api/bce/tasks/{任务 ID}/confirm \
  -H "Content-Type: application/json" \
  -d '{"userName":"匠心"}'
```

---

### 4️⃣ 任务执行

**功能说明：** 负责人执行任务

**状态变更：**
```
待开始 → 进行中 → 已完成/阻塞/待验收
```

**执行要求：**
- 开始执行时更新状态为"进行中"
- 遇到问题标记为"阻塞"并说明原因
- 完成后填写完成说明

**API 示例：**
```bash
# 开始执行
curl -X POST http://localhost:3000/api/bce/tasks/{任务 ID}/start \
  -H "Content-Type: application/json" \
  -d '{"operator":"匠心","comment":"开始执行"}'

# 更新状态
curl -X PUT http://localhost:3000/api/bce/tasks/{任务 ID} \
  -H "Content-Type: application/json" \
  -d '{"status":"executing","userName":"匠心"}'
```

---

### 5️⃣ 任务完成

**功能说明：** 负责人完成任务并提交

**完成操作：**
- OCC 面板点击 [完成] 按钮
- API 提交完成
- 填写完成说明（必填）

**API 示例：**
```bash
curl -X POST http://localhost:3000/api/bce/tasks/{任务 ID}/complete \
  -H "Content-Type: application/json" \
  -d '{"operator":"匠心","comment":"技术方案已完成"}'
```

**自动流转：**
```
完成后 → 规则引擎判断 → 自动流转到下一节点
```

---

### 6️⃣ 任务审核（v3.3 新增驳回）

**功能说明：** 审核节点对任务进行审核

**审核节点：**
- 司库（财务审核）
- 执矩（安全审核）
- 天枢（CEO 验收）

**审核结果：**
| 结果 | 操作 | 流转 |
|------|------|------|
| **通过** | 点击 [通过] | 流转到下一节点 |
| **不通过** | 点击 [驳回] | 回退到上一节点 |

**驳回流程（v3.3 新增）：**
```
1. 审核人点击 [驳回]
2. 填写驳回原因（必填）
3. 系统根据流转规则判断回退节点
4. 任务状态 → 🟢 进行中
5. 通知被驳回人重新执行
6. 记录驳回次数

驳回次数 ≥ 5 次 → 🚨 通知天枢 + 磊哥
```

**API 示例：**
```bash
# 审核通过
curl -X POST http://localhost:3000/api/bce/tasks/{任务 ID}/accept \
  -H "Content-Type: application/json" \
  -d '{"acceptor":"天枢","comment":"验收通过"}'

# 审核驳回
curl -X POST http://localhost:3000/api/bce/tasks/{任务 ID}/reject \
  -H "Content-Type: application/json" \
  -d '{
    "operator":"司库",
    "reason":"预算超支 20%，需要优化方案"
  }'
```

### 7️⃣ 流转规则引擎

**功能说明：** 统一规则引擎，适用于所有流转场景

**规则配置示例：**
```javascript
// src/config/task-rules.js
module.exports = {
  transferRules: [
    {
      name: "技术方案流转",
      match: (task) => task.type === 'technical',
      next: (task) => {
        if (task.assignee === '匠心') return '司库';
        if (task.assignee === '司库') return '执矩';
        if (task.assignee === '执矩') return '天枢';
      }
    },
    {
      name: "财务审核流转",
      match: (task) => task.type === 'finance',
      next: (task) => {
        if (task.assignee === '司库') return '执矩';
        if (task.assignee === '执矩') return '天枢';
      }
    }
  ],
  
  validMembers: ['天枢', '匠心', '司库', '执矩', '磐石', '灵犀', '天策']
};
```

**边界检查：**
- ✅ 不能转交给自己
- ✅ 循环转交检测（≥3 次拦截）
- ✅ 有效成员检查
- ✅ 流转规则统一适用（完成/驳回/转交）

### 8️⃣ 升级通知机制（v3.3 新增）

**功能说明：** 异常情况自动升级通知

**触发条件：**
| 场景 | 阈值 | 通知对象 |
|------|------|---------|
| **驳回次数** | ≥5 次 | 天枢 + 磊哥 |
| **确认超时** | ≥30 分钟 | 创建人 + 磊哥 |
| **执行超时** | 超过 deadline | 创建人 + 磊哥 |

**通知内容：**
```
🚨 任务驳回预警

任务：技术方案设计
驳回次数：5 次（阈值：5）
当前操作人：司库
驳回原因：预算超支

请立即处理！
```

## 📊 任务状态说明

| 状态 | Emoji | 含义 | 可执行操作 |
|------|-------|------|-----------|
| **待开始** | 🟡 | 任务已分配，等待确认 | 确认接收 |
| **进行中** | 🟢 | 已开始执行 | 提交完成/标记阻塞 |

| **待验收** | 🟠 | 任务完成，等待审核 | 审核通过/驳回 |
| **已完成** | ✅ | 验收通过 | 无 |
