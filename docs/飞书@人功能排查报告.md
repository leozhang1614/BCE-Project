## 🔧 可能原因

### 1. 飞书机器人应用权限不足

**检查项：**
- [ ] 机器人是否已添加到群聊
- [ ] 机器人是否有@人的权限
- [ ] 机器人应用是否配置了"发送消息"权限
- [ ] 机器人应用是否配置了"@人"权限

**解决方案：**
1. 登录飞书开放平台
2. 进入应用管理 → 选择 BCE 机器人应用
3. 检查权限配置：
   - ✅ 发送消息权限
   - ✅ 获取用户信息权限
   - ✅ @人权限
4. 重新发布应用

---

### 2. 用户 ID 不匹配

**问题：** 每个飞书应用有独立的 user_id 体系

**当前配置：**
```javascript
'磐石': 'ou_dba586c77d92f652e427370d3f54cc54'
```

**验证方法：**
1. 确认这个 user_id 是当前 BCE 机器人应用中的 ID
2. 不同应用的 user_id 不通用
3. 可能需要重新获取磐石在当前应用中的 user_id

**获取正确 user_id 的方法：**
```bash
# 方法 1: 通过飞书 API 获取
curl -X GET "https://open.feishu.cn/open-apis/contact/v3/users/:user_id" \
  -H "Authorization: Bearer TOKEN"

# 方法 2: 通过群成员列表获取
curl -X GET "https://open.feishu.cn/open-apis/im/v1/chats/CHAT_ID/members" \
  -H "Authorization: Bearer TOKEN"
```

---

### 3. 飞书客户端设置问题

**检查项：**
- [ ] 磐石的飞书是否开启了@通知
- [ ] 是否设置了免打扰
- [ ] 是否在电脑端/手机端都收不到

---

### 4. 消息格式问题

**飞书@人的正确格式：**

#### 文本消息（text）
```json
{
  "receive_id": "chat_id",
  "msg_type": "text",
  "content": "{\"text\":\"消息内容\",\"at\":{\"user_id\":[\"ou_xxx\"],\"all\":false}}"
}
```

#### POST 消息（富文本）
```json
{
  "receive_id": "chat_id",
  "msg_type": "post",
  "content": "{\"zh_cn\":{\"title\":\"标题\",\"content\":[[{\"tag\":\"at\",\"user_id\":\"ou_xxx\"}]]}}"
}
```

**注意：**
- `at` 字段必须在 `content` 内部
- `user_id` 必须是数组格式
- `receive_id_type` 参数必须是 `chat_id`
