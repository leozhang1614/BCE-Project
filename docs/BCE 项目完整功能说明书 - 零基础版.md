# 第一部分：项目入门

## 1. 这个项目是什么？

### 1.1 一句话介绍

**BCE（北斗协同引擎）是一个任务管理和协同系统，帮助团队高效分配任务、跟踪进度、确保任务按时完成。**

### 1.2 形象比喻

想象你有一个**智能任务管家**：

```
你（管理者）说："匠心，去做 BCE 技术方案"
           ↓
任务管家自动：
  1. ✅ 记录这个任务
  2. ✅ 通知匠心
  3. ✅ 提醒匠心确认
  4. ✅ 跟踪任务进度
  5. ✅ 完成后通知你
```

**BCE 就是这个"任务管家"！**

### 1.3 核心能力

| 能力 | 说明 | 例子 |
|------|------|------|
| **任务分配** | 把任务分配给具体的人 | 天枢分配任务给匠心 |
| **及时通知** | 任务变动及时通知相关人员 | 匠心收到飞书通知 |
| **进度跟踪** | 实时查看任务做到哪一步了 | 查看任务看板 |
| **任务流转** | 任务在不同人之间传递 | 匠心→司库→执矩 |
| **完成确认** | 确保任务被确认和执行 | 匠心点击"确认收到" |

## 2. 为什么要做这个项目？

### 2.1 遇到的问题

**场景 1：任务发出去，没人理**
```
天枢："匠心，做这个任务"
匠心：（没看到消息）
天枢：（等了好久）"怎么还没开始？"
匠心："啊？我没收到通知..."
```
❌ **问题：** 任务通知不到位

**场景 2：任务做完了，没人知道**
```
匠心：（默默完成任务）
天枢：（不知道已完成）"任务怎么还没好？"
匠心："我早就做完了啊..."
```
❌ **问题：** 任务状态不透明

**场景 3：多人协作，责任不清**
```
天枢："这个任务谁负责？"
匠心："我以为司库在做"
司库："我以为执矩在处理"
```
❌ **问题：** 任务流转不清晰

### 2.2 BCE 的解决方案

**针对场景 1：双重通知保障**
```
天枢创建任务
     ↓
BCE 系统：
  1. OCC 协作页面显示 ✅
  2. 飞书发送通知 ✅
  3. 每 5 分钟提醒一次 ✅
```
✅ **结果：** 匠心一定能收到通知

**针对场景 2：实时状态同步**
```
匠心完成任务 → 点击 [提交验收]
                    ↓
              OCC 任务页面立即更新
                    ↓
              天枢看到"待验收"状态
```
✅ **结果：** 所有人实时看到任务状态

**针对场景 3：流转历史追踪**
```
OCC 协作页面显示：
  09:00 天枢 → 匠心（任务分配）✅
  09:30 匠心确认接收 ✅
  10:00 匠心 → 司库（协作请求）✅
  10:30 司库提供数据 ✅
```
✅ **结果：** 谁在什么时候做了什么，一目了然

## 3. 项目叫什么名字？

### 3.1 项目名称

**中文名：** 北斗协同引擎  
**英文名：** Beidou Collaboration Engine  
**简称：** BCE

### 3.2 名字含义

- **北斗：** 指引方向，像北斗星一样指引任务流向
- **协同：** 多人协作，任务在不同人之间流转
- **引擎：** 驱动任务自动流转，无需人工催促

### 3.3 相关系统

| 系统 | 关系 | 说明 |
|------|------|------|
| **BCE** | 主项目 | 任务管理和通知系统 |
| **OCC** | 合作伙伴 | OpenClaw 控制中心，展示任务 |
| **飞书** | 通知通道 | 发送消息提醒 |
| **OpenClaw** | 底层平台 | 提供会话管理能力 |

## 6. 每个功能怎么用？

### 6.1 任务创建

**谁用：** 天枢（管理者）  
**什么时候用：** 有新工作需要分配时

**使用步骤：**

**方式 A：通过 OpenClaw 创建（推荐）**
```
步骤 1：天枢在自己的 OpenClaw 会话中输入

  sessions_send({
    sessionKey: 'agent-jiangxin',
    message: `
📋 新任务分配

【任务名称】BCE 技术方案设计
【优先级】P0
【截止时间】今天 18:00
【任务描述】设计 BCE 项目技术架构
`
  })

步骤 2：发送

结果：✅ 任务创建成功，匠心收到通知
```

**方式 B：通过 OCC 页面创建**
```
步骤 1：访问 OCC 任务页面
  http://192.168.31.187:4310/?section=tasks

步骤 2：点击 [新建任务] 按钮

步骤 3：填写任务信息
  - 任务名称：BCE 技术方案设计
  - 负责人：匠心
  - 优先级：P0
  - 截止时间：2026-03-21 18:00

步骤 4：点击 [提交]

结果：✅ 任务创建成功
```

**方式 C：通过 API 创建**
```bash
curl -X POST http://localhost:3000/api/bce/tasks \
  -H "Content-Type: application/json" \
  -d '{
    "title": "BCE 技术方案设计",
    "assignee": "匠心",
    "priority": "P0"
  }'
```

**谁用：** 匠心（任务负责人）  
**什么时候用：** 收到新任务通知时

```
结果：✅ 任务完成流转
```

```
┌─────────────────────────────────────────────────────────────────┐
│                         用户看到的界面                           │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  ┌──────────────┐  ┌──────────────┐  ┌──────────────┐          │
│  │  OCC 协作页面 │  │  OCC 任务页面 │  │  飞书群聊    │          │
│  │              │  │              │  │              │          │
│  │  查看任务    │  │  管理任务    │  │  接收通知    │          │
│  └──────┬───────┘  └──────┬───────┘  └──────┬───────┘          │
│         │                 │                 │                   │
│         └─────────────────┼─────────────────┘                   │
│                           │                                     │
│                           ↓                                     │
│              ┌────────────────────────┐                        │
│              │    OpenClaw Gateway    │                        │
│              │   (会话管理中樞)        │                        │
│              └───────────┬────────────┘                        │
│                          │                                      │
│                          ↓                                      │
│              ┌────────────────────────┐                        │
│              │      BCE 系统          │                        │
│              │  (任务管理 + 通知)      │                        │
│              └────────────────────────┘                        │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```

### 9.2 数据流向

```
用户操作 → 前端页面 → API 接口 → 业务逻辑 → 数据存储
   ↓                                          ↓
通知发送 ← 通知服务 ← 状态变更 ← 更新数据 ← 查询数据
```

**具体例子：天枢创建任务**

```
1. 天枢在 OpenClaw 输入创建任务指令
   ↓
2. OpenClaw Gateway 接收请求
   ↓
3. 转发给 BCE 系统
   ↓
4. BCE 调用任务管理 API（bce-tasks.js）
   ↓
5. 任务服务创建任务并保存（bce-data.json）
   ↓
6. 通知服务发送飞书通知（feishu-notify.js）
   ↓
7. 同步服务同步到 OCC（occ-sync.js）⏳待开发
   ↓
8. 匠心在 OCC 看到新任务
```

## 15. 常见问题解答

### Q1：收不到飞书通知怎么办？

**可能原因：**
1. BCE 服务未运行
2. 定时任务未启动
3. 飞书配置错误

**解决方法：**
```bash
# 1. 检查 BCE 服务
curl http://localhost:3000/health

# 2. 查看日志
tail -f logs/scheduler.log

# 3. 检查飞书配置
cat .env | grep FEISHU
```

### Q2：OCC 页面不显示任务怎么办？

**可能原因：**
1. OCC 服务未运行
2. OCC 与 Gateway 连接断开
3. 浏览器缓存问题

**解决方法：**
```bash
# 1. 检查 OCC 服务
curl http://192.168.31.187:4310/api/tasks

# 2. 刷新页面（Ctrl+F5 强制刷新）

# 3. 清除浏览器缓存
```

### Q3：任务确认失败怎么办？

**可能原因：**
1. 任务 ID 错误
2. API 请求格式错误
3. 任务不存在

**解决方法：**
```bash
# 1. 检查任务 ID
curl http://localhost:3000/api/bce/tasks

# 2. 查看 BCE 日志
tail -f logs/bce.log

# 3. 重新确认
```
