## 1. 制度概述

### 1.1 制度目的

**核心目标：**
- ✅ 规范任务节点跟进流程
- ✅ 明确各角色职责
- ✅ 保证任务按时高质量完成
- ✅ 实现进度透明可控
- ✅ 建立有效的催促和问责机制

**解决的问题：**
- ❌ 任务进度不透明
- ❌ 节点跟进不及时
- ❌ 责任不明确
- ❌ 催促无实质效果
- ❌ 延期无问责

### 1.2 适用范围

**适用对象：**
- 公司所有项目任务
- 所有项目成员（执行者、管理者、CEO）

**适用场景：**
- 项目开发任务
- 技术方案设计
- 文档编写任务
- 运维任务
- 其他需要跟进的工作

### 1.3 核心原则

**透明可控原则：**
- 所有任务必须在 BCE 系统创建
- 所有进度必须实时更新
- 所有延期必须说明原因
- 所有操作必须有记录

**人性化原则：**
- 减少无效催促
- 增加实质支持
- 正向激励为主
- 风险提前预警

**责任明确原则：**
- 每个任务有明确执行者
- 每个任务有明确管理者
- 每个节点有明确截止时间
- 每个延期有明确责任人

**CEO 职责：**
1. ✅ 最终决策权
2. ✅ 资源协调
3. ✅ 重大风险处理
4. ✅ 绩效评定
5. ✅ 制度执行监督

**验收者职责：**
1. ✅ 验收任务完成质量
2. ✅ 提出修改意见
3. ✅ 确认验收通过
4. ✅ 记录验收结果

**审核者职责：**
1. ✅ 审核代码/方案质量
2. ✅ 检查合规性
3. ✅ 提出改进建议
4. ✅ 确认审核通过

## 3. 任务流转流程

### 3.1 标准流程

```
任务创建 → 任务分配 → 确认接收 → 执行任务 → 提交验收 → 验收通过 → 任务完成
                                    ↓
                              进度反馈（每 30 分钟）
                                    ↓
                              风险报告（如有）
```

### 3.2 开发任务流程（带审核）

```
任务创建 → 任务分配 → 确认接收 → 开发 → 提交验收 → 验收通过 → 提交审核 → 审核通过 → 任务完成
                                    ↓                    ↓
                              进度反馈              验收不通过
                                    ↓                    ↓
                              风险报告              回退到开发节点
```

### 3.3 节点定义

| 节点 | 说明 | 时限 | 负责人 |
|------|------|------|--------|
| 待开始 | 任务已分配，等待执行 | - | 执行者 |
| 进行中 | 开始执行任务 | 按 deadline | 执行者 |
| 待验收 | 任务完成，等待验收 | 2 小时 | 验收者 |
| 待审核 | 验收通过，等待审核 | 4 小时 | 审核者 |
| 已完成 | 审核通过，任务完成 | - | - |
| 已驳回 | 验收/审核不通过 | - | 执行者 |

### 3.4 节点流转规则

**正常流转：**
```
待开始 → 执行者确认 → 进行中 → 执行者提交 → 待验收 → 验收者通过 → 待审核 → 审核者通过 → 已完成
```

**驳回流转：**
```
待验收 → 验收者驳回 → 进行中（重新开发）
待审核 → 审核者驳回 → 进行中（重新开发）
```

**超时流转：**
```
待验收（超时 2 小时）→ 自动通知管理者
待审核（超时 4 小时）→ 自动通知 CEO
```

## 5. 进度反馈机制

### 5.1 强制进度反馈（30 分钟）

**触发条件：**
- 执行者确认接收任务后启动
- 每 30 分钟自动触发

**反馈内容：**
- 完成百分比
- 当前工作内容
- 下一步计划
- 剩余时间
- 状态（正常/风险/阻塞）

**超时升级机制：**

| 超时时长 | 通知对象 | 通知方式 | 升级级别 |
|----------|----------|----------|----------|
| 30 分钟 | 执行者 | 飞书消息 | L1 提醒 |
| 60 分钟 | 管理者 | 飞书消息 + @ | L2 警告 |
| 120 分钟 | CEO | 飞书消息 + @ + 电话 | L3 严重 |

### 5.2 标准化进度汇报（1.5 小时）

**触发条件：**
- 每 1.5 小时自动触发
- 仅针对进行中的任务

**汇报内容：**
- 完成百分比
- 已完成工作列表
- 剩余工作列表
- 预计完成时间
- 风险/阻塞说明

**绩效积分：**
- 按时汇报：+2 分
- 超时汇报：-5 分
- 高质量汇报（管理者标记）：+3 分

### 5.3 风险报告机制

**风险等级：**

| 等级 | 颜色 | 触发条件 | 通知对象 |
|------|------|----------|----------|
| 正常 | 🟢 | 进度正常，无超时 | 无 |
| 关注 | 🔵 | 进度落后<20% | 执行者 |
| 警告 | 🟠 | 进度落后 20-50% 或超时<60 分钟 | 执行者 + 管理者 |
| 高风险 | 🔴 | 进度落后>50% 或超时≥60 分钟 | 执行者 + 管理者 + CEO |
| 严重 | ⚫ | 失联>4 小时或延期>24 小时 | 全员 + 电话通知 |

**风险报告流程：**
```
执行者发现风险
    ↓
在 BCE 系统标记风险等级
    ↓
填写风险说明和建议方案
    ↓
系统自动通知相关人员
    ↓
管理者协调处理（30 分钟内）
    ↓
风险解除或升级
```

## 6. 管理者权限

### 6.1 管理者实质性权力

**✅ 重新分配任务**
```
权力：可将任务从执行者 A 转给执行者 B
触发：执行者延期/能力不足/优先级调整
绩效影响：
  - 原执行者：-15 分
  - 新执行者：接收任务，正常计分
```

**✅ 调整优先级**
```
权力：高/中/低优先级调整
影响：
  - 高优先级：汇报频率×2，提醒更频繁
  - 中优先级：标准频率
  - 低优先级：汇报频率÷2，减少打扰
```

**✅ 调整截止时间**
```
权力：延长或缩短任务 deadline
限制：
  - 延长：最多延长原 deadline 的 50%
  - 缩短：需执行者确认（可拒绝）
  - 超过限制：需 CEO 批准
```

**✅ 强制汇报**
```
权力：随时要求执行者汇报进度
限制：每任务每日≤3 次（避免滥用）
流程：
  1. 管理者点击"强制汇报"
  2. 执行者收到飞书消息（高优先级）
  3. 执行者 10 分钟内回复进度
  4. 超时自动上报 CEO
绩效影响：
  - 按时回复：无影响
  - 超时回复：-5 分
```

**✅ 标记风险**
```
权力：将任务标记为🔴高风险
触发条件：
  - 执行者多次超时
  - 技术难度超出预期
  - 资源不足
影响：
  - CEO 自动收到通知
  - 任务在仪表盘高亮显示
  - 管理者需提交风险缓解方案
```

**✅ 上报 CEO**
```
权力：将严重问题上报天枢
触发条件：
  - 任务严重延期（>24 小时）
  - 执行者失联（>4 小时）
  - 需要跨部门协调
  - 需要额外资源
流程：
  1. 管理者提交上报申请
  2. 附带问题描述和建议方案
  3. CEO 收到通知并决策
```

### 6.2 权限使用规范

**使用原则：**
- 谨慎使用，避免滥用
- 记录原因，接受监督
- 公平公正，一视同仁

**禁止行为：**
- ❌ 无理由重新分配任务
- ❌ 频繁调整优先级（>3 次/日）
- ❌ 恶意缩短截止时间
- ❌ 滥用强制汇报（>3 次/日）

**监督机制：**
- 所有操作记录审计日志
- CEO 可随时查看
- 执行者可申诉

## 7. 绩效考核

### 7.1 积分规则

**正向激励：**
| 行为 | 积分 | 说明 |
|------|------|------|
| 准时完成 | +10 | 在截止时间前完成 |
| 提前完成（>24h） | +20 | 提前 24 小时以上 |
| 提前完成（>48h） | +30 | 提前 48 小时以上 |
| 每次进度汇报 | +2 | 1.5 小时模板汇报 |
| 高质量汇报 | +3 | 管理者标记为"优秀" |
| 主动承担额外任务 | +5 | 自愿接手 |
| 帮助他人完成任务 | +10 | 协作奖励 |

**负向约束：**
| 行为 | 积分 | 说明 |
|------|------|------|
| 延期完成 | -10 | 超过截止时间 |
| 严重延期（>24h） | -20 | 超过 24 小时 |
| 未按时汇报 | -5 | 超时未反馈 |
| 被重新分配 | -15 | 管理者强制转交 |
| 标记高风险（个人原因） | -10 | 因个人原因导致 |
| 失联（>4h） | -20 | 无法联系 |
| 质量不达标被驳回 | -10 | 审核不通过 |

### 7.2 等级评定

**实时等级：**
| 等级 | 积分范围 | 奖金系数 | 特权 |
|------|----------|----------|------|
| S | ≥100 | 1.5x | 优先选择任务 |
| A | 80-99 | 1.2x | 延长 deadline 50% |
| B | 60-79 | 1.0x | 标准权限 |
| C | 40-59 | 0.8x | 需管理者审核 |
| D | <40 | 0.5x | 限制任务接收 |

**等级更新频率：**
- 实时更新积分
- 每日凌晨 02:00 重新评定等级
- 等级变更通知本人 + 管理者

### 7.3 绩效看板

**展示内容：**
- 个人积分和等级
- 团队排名
- 积分趋势图
- 任务完成率
- 准时完成率

**访问权限：**
- 个人：查看自己的绩效
- 管理者：查看团队成员绩效
- CEO：查看全员绩效

## 8. 违规处理

### 8.1 违规等级

| 等级 | 违规行为 | 处理方式 |
|------|----------|----------|
| 一级 | 超时未确认、未反馈 | 警告 |
| 二级 | 不走工作流、不验收 | 通报批评 |
| 三级 | 影响系统稳定、绕过审核 | 严重警告 + 绩效扣分 |

### 8.2 处理流程

```
发现违规
    ↓
记录违规事实
    ↓
通知违规者
    ↓
违规者申诉（可选，24 小时内）
    ↓
管理者决定处理方式
    ↓
执行处理
    ↓
记录归档
```

### 8.3 申诉机制

**申诉条件：**
- 对违规事实有异议
- 有正当理由说明

**申诉流程：**
```
提交申诉申请
    ↓
管理者审核（24 小时内）
    ↓
CEO 终审（如需要）
    ↓
维持或撤销处理
```
