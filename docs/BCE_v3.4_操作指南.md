## 执行者操作指南

### 你是执行者吗？

如果你被分配了任务，你就是执行者。

### 你的职责

1. ✅ **确认任务** - 收到任务后 30 分钟内确认
2. ✅ **更新进度** - 每 30 分钟更新一次进度百分比
3. ✅ **按模板汇报** - 收到汇报提醒后按要求回复
4. ✅ **遇到困难立即报告** - 不要等到超时

---

### 操作流程

#### 1️⃣ 接收任务

**你会收到飞书通知：**

```
📋 新任务分配

任务：北斗智投系统 - 智能决策模块
描述：开发智能决策引擎
优先级：P1
截止时间：2026-03-24 17:00

[确认接收] [需要沟通]
```

**你应该：**
- ✅ 点击"确认接收"
- ⏰ 设置第一个 30 分钟后的提醒
- 📝 开始工作

---

#### 2️⃣ 更新进度（每 30 分钟）

**方式一：飞书卡片更新**

收到进度提醒时：

```
⏰ 进度更新提醒

任务：北斗智投系统 - 智能决策模块
当前进度：0%
最后更新：刚刚

⚠️ 已超过 30 分钟未更新进度，请立即更新！

[立即更新] [稍后提醒]
```

点击"立即更新"，填写：

```
完成百分比：50%
已完成：数据采集模块、数据清洗模块
剩余工作：三维分析、建议生成
预计完成：2026-03-24 17:00
风险/阻塞：无
```

**方式二：API 更新（技术人员）**

```bash
curl -X POST http://localhost:3000/api/bce/progress/<TASK_ID> \
  -H "Content-Type: application/json" \
  -d '{
    "percent": 50,
    "completed": "数据采集模块",
    "remaining": "三维分析",
    "estimate": "2026-03-24 17:00",
    "operator": "你的名字"
  }'
```

---

#### 3️⃣ 标准化汇报（每 1.5 小时）

**你会收到汇报模板：**

```
📋 定时进度汇报

任务：北斗智投系统 - 智能决策模块
负责人：匠心
截止时间：2026-03-24 17:00
当前状态：🚀 executing

━━━━━━━━━━━━━━━━━━━━

✅ 完成百分比：50%

✅ 已完成：
   - 数据采集模块
   - 数据清洗模块

✅ 剩余工作：
   - 三维分析
   - 建议生成

✅ 预计完成：2026-03-24 17:00

✅ 风险/阻塞：无

━━━━━━━━━━━━━━━━━━━━

请在 30 分钟内回复更新！
[立即更新] [请求帮助]
```

**你应该：**
- ✅ 按实际进度填写
- ✅ 30 分钟内回复
- ⚠️ 有困难点"请求帮助"

---

#### 4️⃣ 遇到困难

**不要等到超时！立即报告！**

```
🆘 请求帮助

任务：北斗智投系统 - 智能决策模块
问题：三维分析模块遇到技术难点
影响：可能延期 2 小时
需要：磐石协助

[发送]
```

**好处：**
- ✅ 管理者可以立即介入
- ✅ 可以重新分配或调整时间
- ✅ 不会影响绩效（主动报告）

---

#### 5️⃣ 完成任务

**进度达到 100% 后：**

```
✅ 申请验收

任务：北斗智投系统 - 智能决策模块
完成说明：
  1. 数据采集模块 - 已完成
  2. 数据清洗模块 - 已完成
  3. 三维分析模块 - 已完成
  4. 建议生成模块 - 已完成

测试报告：已通过
文档：已更新

[提交验收]
```

---

### 绩效计算

**你的绩效这样计算：**

```
基础分：
  准时完成：+10 分
  提前 2 小时以上：+20 分
  延期：-10 分（每小时 -5 分）

过程分：
  每次主动汇报：+2 分
  每次未汇报：-5 分

影响分：
  被重新分配：-15 分
  高风险标记：-10 分

质量分：
  验收 5 分：+10 分
  验收 4 分：+5 分
  验收 3 分：0 分
  验收 2 分：-5 分
  验收 1 分：-10 分
```

**最终等级：**

| 总分 | 等级 | 奖金系数 |
|------|------|----------|
| ≥50 | S（卓越） | 1.2 |
| 30-49 | A（优秀） | 1.1 |
| 10-29 | B（良好） | 1.0 |
| 0-9 | C（需改进） | 0.9 |
| <0 | D（不达标） | 0.7 |

---

### 预警升级流程

**如果你不更新进度：**

```
0 分钟：你确认任务
  ↓
30 分钟：你收到提醒（飞书）
  ↓
60 分钟：管理者收到通知（建议介入）
  ↓
120 分钟：CEO 收到报告（标记高风险）
```

**后果：**
- 30 分钟：只是提醒，无影响
- 60 分钟：管理者可能介入
- 120 分钟：CEO 知道，绩效 -10 分

**建议：** 定个 25 分钟的闹钟！

## 管理者操作指南

### 你是管理者吗？

如果你是项目发起人或 CEO 指定的负责人，你就是管理者。

### 你的权力

1. 👁️ **查看所有任务** - 实时进度仪表盘
2. 🔄 **重新分配任务** - 执行者不力时换人
3. ⚡ **调整优先级** - P0/P1/P2 动态调整
4. 📅 **调整截止时间** - 根据业务需求
5. 🔔 **强制要求汇报** - 紧急情况下
6. 🚨 **标记风险** - 高亮问题任务
7. 📢 **上报 CEO** - 需要决策时

#### 2️⃣ 重新分配任务

**何时使用：**
- 执行者进度严重滞后
- 执行者遇到无法克服的困难
- 业务优先级变化

**操作：**

```bash
curl -X POST http://localhost:3000/api/bce/manager/<TASK_ID>/reassign \
  -H "Content-Type: application/json" \
  -d '{
    "newAssignee": "磐石",
    "reason": "原执行者进度滞后，预计无法按时交付",
    "handover": "请保留已完成的数据采集代码",
    "manager": "司库"
  }'
```

**系统自动执行：**
1. ✅ 通知原执行者（绩效 -15 分）
2. ✅ 通知新执行者（新任务分配）
3. ✅ 通知 CEO（备案）
4. ✅ 更新任务历史

---

#### 3️⃣ 调整优先级

**何时使用：**
- CEO 要求优先交付
- 业务需求变化
- 依赖任务完成

**操作：**

```bash
curl -X POST http://localhost:3000/api/bce/manager/<TASK_ID>/priority \
  -H "Content-Type: application/json" \
  -d '{
    "priority": "P0",
    "reason": "CEO 要求优先交付",
    "manager": "司库"
  }'
```

**效果：**
- ✅ 执行者收到通知
- ✅ P0 任务 30 分钟未更新即上报 CEO
- ✅ 在仪表盘高亮显示

---

    "reason": "业务需求提前",

**效果：**

- ✅ 执行者收到通知
- ✅ 调整预警阈值

---

#### 5️⃣ 强制要求汇报

#### 6️⃣ 标记风险

**何时使用：**
- 进度滞后超过 50%
- 技术难点无法解决
- 资源不足

**操作：**

```bash
curl -X POST http://localhost:3000/api/bce/manager/<TASK_ID>/mark-risk \
  -H "Content-Type: application/json" \
  -d '{
    "level": "high",
    "reason": "进度滞后 50%，预计延期 2 小时",
    "action": "已安排磐石协助",
    "manager": "司库"
  }'
```

**效果：**
- ✅ 任务在仪表盘高亮（红色）
- ✅ CEO 收到通知
- ✅ 记录管理干预

---

#### 7️⃣ 上报 CEO

**何时使用：**
- 需要 CEO 决策
- 需要额外资源
- 需要调整范围

**操作：**

```bash
curl -X POST http://localhost:3000/api/bce/manager/<TASK_ID>/escalate \
  -H "Content-Type: application/json" \
  -d '{
    "reason": "任务严重滞后，需要 CEO 决策",
    "suggestions": [
      "延期交付",
      "增加人手",
      "削减功能"
    ],
    "impact": "影响北斗智投整体上线",
    "manager": "司库"
  }'
```

**效果：**
- ✅ CEO 收到详细报告
- ✅ 包含任务历史、进度曲线
- ✅ 等待 CEO 决策

### 管理建议

#### 最佳实践

1. **每天早中晚查看仪表盘**
   - 09:00 查看整体进度
   - 13:00 识别风险任务
   - 18:00 检查当日完成情况

2. **主动介入，不要等待**
   - 发现进度滞后立即联系
   - 提供帮助而不是指责
   - 必要时重新分配

3. **合理使用权力**
   - 不要频繁调整优先级
   - 重新分配要谨慎
   - 上报 CEO 要有充分理由

4. **记录管理决策**
   - 所有操作都有记录
   - 便于事后复盘
   - 保护自己和团队

## 常见问题 FAQ

### Q1: 忘记更新进度怎么办？

**A:** 立即更新！系统只惩罚不更新，不惩罚晚更新。

```
忘记更新 → 立即补上 → 系统重置计时器 → 无绩效影响
```

**建议：** 定个 25 分钟的闹钟！

---

### Q2: 我真的没时间更新，一直在编码！

**A:** 理解，但进度可见性很重要。

**解决方案：**
1. 花 1 分钟更新进度百分比
2. 简单填写"功能开发中"
3. 系统知道你在工作，不会打扰

---

### Q3: 遇到困难，应该先尝试还是立即报告？

**A:** 立即报告！

```
立即报告 → 管理者介入 → 快速解决 → 无绩效影响
自己硬扛 → 超时未更新 → 预警升级 → 绩效扣分
```

**记住：** 报告困难不是能力问题，是职业素养！

---

### Q4: 被重新分配任务，绩效一定会扣分吗？

**A:** 是的，-15 分。

**但有两种情况：**
1. **执行者原因**（进度滞后、能力不足）- 扣分合理
2. **业务原因**（优先级变化、资源调整）- 可以申诉

**申诉方式：** 向 CEO 说明情况，可申请撤销扣分。

---

### Q5: 如何查看我的绩效？

**A:** 两种方式：

**方式 1：API 查询**
```bash
curl http://localhost:3000/api/bce/progress/performance/你的名字
```

**方式 2：飞书机器人**
```
@BCE 机器人 我的绩效
```

---

### Q6: 绩效等级多久计算一次？

**A:** 实时计算，每周汇总。

- **实时：** 每次任务完成后立即计算
- **汇总：** 每周一生成周报，发送 CEO 和管理者

---

### Q7: 预警升级可以关闭吗？

**A:** 不可以，这是系统核心功能。

**但可以避免：**
- 每 30 分钟更新一次进度
- 遇到困难立即报告
- 完成任务及时提交

---

### Q8: 我是管理者，可以修改自己的任务吗？

**A:** 可以，但不建议。

**建议做法：**
- 让自己的任务由其他管理者监督
- 避免利益冲突
- 保持透明度

---

### Q9: 系统支持多人协作一个任务吗？

**A:** 当前版本一个任务一个执行者。

**多人协作方案：**
1. 拆分子任务
2. 每个人负责一个子任务
3. 主任务由管理者协调

---
