## 1. 快速开始

**一键安装脚本（推荐）：**

```bash
# macOS / Linux
curl -fsSL https://raw.githubusercontent.com/leozhang1614/BCE-Project/main/scripts/install.sh | bash

# 或者使用 git clone
git clone https://github.com/leozhang1614/BCE-Project.git
cd BCE-Project
./scripts/install.sh
```

**预计时间：** 5-10 分钟

## 2. 前置要求

### 2.1 系统要求

| 系统 | 版本要求 | 说明 |
|------|----------|------|
| macOS | 10.15+ | 推荐 macOS 12+ |
| Linux | Ubuntu 20.04+ | 或其他 Debian/RedHat 系 |
| Windows | WSL2 | 推荐使用 WSL2 |

### 2.2 软件要求

**必需软件：**
- ✅ Node.js 18.x 或更高版本
- ✅ npm 9.x 或更高版本
- ✅ Git
- ✅ Docker（可选，用于容器化部署）

**可选软件：**
- 📝 PM2（进程管理）
- 📝 Nginx（反向代理）

### 2.3 检查环境

```bash
# 检查 Node.js 版本
node -v
# 应输出：v18.x.x 或更高

# 检查 npm 版本
npm -v
# 应输出：9.x.x 或更高

# 检查 Git
git --version
# 应输出：git version 2.x.x

# 检查 Docker（可选）
docker --version
# 应输出：Docker version 20.x.x
```

**如未安装 Node.js：**

```bash
# macOS (使用 Homebrew)
brew install node@18

# Ubuntu/Debian
curl -fsSL https://deb.nodesource.com/setup_18.x | sudo -E bash -
sudo apt-get install -y nodejs

# CentOS/RHEL
curl -fsSL https://rpm.nodesource.com/setup_18.x | sudo bash -
sudo yum install -y nodejs
```

## 3. 安装步骤

### 3.1 方案 A：源码安装（推荐）

**步骤 1：克隆仓库**

```bash
# 选择安装目录
cd ~

# 克隆仓库
git clone https://github.com/leozhang1614/BCE-Project.git

# 进入项目目录
cd BCE-Project
```

**步骤 2：安装依赖**

```bash
# 安装 Node.js 依赖
npm install

# 预计时间：2-5 分钟（取决于网络）
```

**步骤 3：配置环境变量**

```bash
# 复制环境配置模板
cp .env.example .env

# 编辑配置文件
nano .env
# 或使用其他编辑器：vim .env / code .env
```

**步骤 4：初始化数据库**

```bash
# 创建 SQLite 数据库（自动）
npm run db:init

# 或使用 Docker（可选）
docker-compose up -d postgres
```

**步骤 5：启动服务**

```bash
# 开发模式（带热重载）
npm run dev

# 生产模式
npm start

# 后台运行（使用 PM2）
pm2 start npm --name "bce" -- start
```

---

### 3.2 方案 B：Docker 安装（最简单）

**步骤 1：安装 Docker**

```bash
# macOS
# 下载 Docker Desktop: https://www.docker.com/products/docker-desktop

# Ubuntu
sudo apt-get update
sudo apt-get install -y docker.io docker-compose

# 启动 Docker
sudo systemctl start docker
sudo systemctl enable docker
```

**步骤 2：拉取镜像**

```bash
cd BCE-Project

# 从 Docker Hub 拉取（如已发布）
docker pull leozhang1614/bce-project:latest

# 或本地构建
docker-compose build
```

**步骤 3：启动容器**

```bash
# 使用 docker-compose
docker-compose up -d

# 查看运行状态
docker-compose ps

# 查看日志
docker-compose logs -f
```

**步骤 4：访问服务**

```
http://localhost:3000
```

## 4. 配置说明

### 4.1 环境配置文件

**.env 文件位置：**
```
/BCE-Project/.env
```

**配置模板：**
```bash
# ==================== 应用配置 ====================
NODE_ENV=production
PORT=3000
HOST=0.0.0.0

# ==================== 数据库配置 ====================
DB_TYPE=sqlite
# SQLite 配置（默认）
DB_PATH=./data/bce.db

# PostgreSQL 配置（可选）
# DB_TYPE=postgres
# DB_HOST=localhost
# DB_PORT=5432
# DB_NAME=bce
# DB_USER=bce_user
# DB_PASSWORD=your_password

# ==================== 飞书配置 ====================
FEISHU_APP_ID=cli_xxxxxxxxxxxxx
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxxxxx
FEISHU_APP_TICKET=xxxxxxxxxxxxxxxxxxxxx

# ==================== OpenClaw 配置 ====================
OPENCLAW_WORKSPACE=/Users/ai/.openclaw/workspace-jiangxin
OPENCLAW_CHANNEL=feishu
OPENCLAW_CHAT_ID=oc_xxxxxxxxxxxxx

# ==================== 定时任务配置 ====================
ENABLE_SCHEDULER=true
PROGRESS_CHECK_INTERVAL=1800000  # 30 分钟
REPORT_INTERVAL=5400000          # 1.5 小时

# ==================== 日志配置 ====================
LOG_LEVEL=info
LOG_FILE=./logs/bce.log
LOG_MAX_SIZE=10MB
LOG_MAX_FILES=7

# ==================== 性能配置 ====================
CACHE_ENABLED=true
CACHE_TTL=300  # 5 分钟
```

### 4.2 关键配置说明

**飞书配置获取：**

1. 访问 https://open.feishu.cn/app
2. 创建企业自建应用
3. 获取 App ID 和 App Secret
4. 配置事件订阅（接收消息）
5. 配置机器人功能（发送消息）

**OpenClaw 配置获取：**

1. 查看 OpenClaw 配置文件
2. 找到 workspace 路径
3. 找到默认 channel 和 chat_id

**端口配置：**

- 默认端口：3000
- 如端口冲突，修改 `PORT=xxxx`
- 确保防火墙开放该端口

## 5. 启动与验证

### 5.1 启动服务

```bash
# 方式 1：前台运行（开发调试）
npm run dev

# 方式 2：后台运行（生产环境）
npm start

# 方式 3：使用 PM2（推荐生产环境）
pm2 start npm --name "bce" -- start
pm2 save
pm2 startup
```

### 5.2 验证安装

**检查 1：服务状态**

```bash
# 检查进程
ps aux | grep node

# 检查端口
lsof -i :3000

# 或
netstat -tlnp | grep 3000
```

**检查 2：API 健康检查**

```bash
curl http://localhost:3000/api/health

# 应返回：
# {"status":"healthy","timestamp":"2026-03-23T10:00:00Z"}
```

**检查 3：访问管理仪表盘**

```
http://localhost:3000/dashboard
```

应看到：
- ✅ 任务列表
- ✅ 进度面板
- ✅ 绩效排行榜

**检查 4：飞书集成测试**

```bash
# 发送测试消息
curl -X POST http://localhost:3000/api/test/feishu \
  -H "Content-Type: application/json" \
  -d '{"message":"BCE 系统测试"}'
```

应在飞书收到测试消息。

### 5.3 日志检查

```bash
# 查看实时日志
tail -f logs/bce.log

# 查看错误日志
grep ERROR logs/bce.log | tail -20

# 使用 PM2 查看日志
pm2 logs bce
```

**正常日志示例：**
```

[INFO] Database initialized: sqlite
```

## 6. 常见问题

### 6.1 安装问题

**Q1: npm install 失败**

```bash
# 清除 npm 缓存
npm cache clean --force

# 删除 node_modules 重新安装
rm -rf node_modules package-lock.json
npm install

# 使用淘宝镜像（国内）
npm config set registry https://registry.npmmirror.com
npm install
```

**Q2: Git 克隆失败**

```bash
# 检查网络连接
ping github.com

# 使用 HTTPS 代替 SSH
git clone https://github.com/leozhang1614/BCE-Project.git

# 或使用镜像
git clone https://gitee.com/mirror/BCE-Project.git
```

**Q3: 端口被占用**

```bash
# 查找占用端口的进程
lsof -i :3000

# 杀死进程
kill -9 <PID>

# 或修改配置使用其他端口
PORT=3001
```

### 6.2 运行问题

**Q4: 启动后闪退**

```bash
# 查看详细错误
npm run dev

# 检查日志
cat logs/bce.log | tail -50

# 常见原因：
# 1. 配置文件缺失 → 复制 .env.example
# 2. 数据库权限问题 → chmod 755 data/
# 3. 端口冲突 → 修改 PORT
```

**Q5: 飞书消息无法发送**

```bash
# 检查飞书配置
cat .env | grep FEISHU

# 测试飞书 API
curl -X POST https://open.feishu.cn/open-apis/auth/v3/tenant_access_token/internal \
  -H "Content-Type: application/json" \
  -d '{"app_id":"xxx","app_secret":"xxx"}'

# 检查网络连接
ping open.feishu.cn
```

**Q6: 定时任务不执行**

```bash
# 检查 Scheduler 是否启用
cat .env | grep SCHEDULER

# 查看定时任务日志
grep "scheduler" logs/bce.log

# 重启服务
pm2 restart bce
```

### 6.3 性能问题

**Q7: 系统响应慢**

```bash
# 检查 CPU 使用率
top -pid $(pgrep -f "node.*bce")

# 检查内存使用
pm2 monit

# 优化建议：
# 1. 启用缓存 CACHE_ENABLED=true
# 2. 增加 Node.js 内存 NODE_OPTIONS="--max-old-space-size=4096"
# 3. 使用 PM2 集群模式 pm2 start npm -i 4
```

**Q8: 数据库查询慢**

```bash
# 检查数据库大小
ls -lh data/bce.db

# 添加索引（如数据量大）
sqlite3 data/bce.db "CREATE INDEX IF NOT EXISTS idx_tasks_status ON tasks(status);"

# 或迁移到 PostgreSQL
```

### 6.4 升级问题

**Q9: 如何升级到新版本**

```bash
# 备份当前版本
cp -r BCE-Project BCE-Project.backup

# 拉取最新代码
cd BCE-Project
git pull origin main

# 安装新依赖
npm install

# 执行数据库迁移
npm run db:migrate

# 重启服务
pm2 restart bce

# 验证版本
curl http://localhost:3000/api/version
```

## 7. 卸载方法

### 7.1 完全卸载

```bash
# 停止服务
pm2 stop bce
pm2 delete bce

# 删除项目目录
cd ~
rm -rf BCE-Project

# 删除数据库
rm -rf ~/.bce/data

# 删除日志
rm -rf ~/.bce/logs

# 删除配置
rm -rf ~/.bce/config

# 清理 PM2
pm2 save

# 可选：删除 Node.js（如仅用于 BCE）
# brew uninstall node  # macOS
# sudo apt-get remove nodejs  # Ubuntu
```

### 7.2 保留数据卸载

```bash
# 备份数据
cp -r BCE-Project/data ~/bce-data-backup
cp -r BCE-Project/logs ~/bce-logs-backup
cp BCE-Project/.env ~/bce-config-backup.env

# 删除项目
rm -rf BCE-Project

# 恢复时：
# git clone ...
# cp ~/bce-data-backup BCE-Project/data
# cp ~/bce-config-backup.env BCE-Project/.env
```
