## ⚠️ 发现的问题

### 问题 1：OCC 历史任务无法批量清除

**问题描述：**
- OCC 系统没有提供批量删除 API
- 137 个未完成历史任务无法清除
- 只能手动逐个删除

**影响：**
- 历史任务保留在 OCC 系统
- 新任务通过 BCE v3.2 创建，不冲突
- 影响：低

**根本原因：**
- OCC 设计为只读展示层
- 不支持任务删除操作
- 需要通过 OpenClaw sessions_send 或手动删除

**解决方案：**

**方案 A（推荐）：忽略历史任务**
- 保留作为历史记录
- 新任务从 BCE v3.2 开始
- 在培训公告中说明

**方案 B：手动清理**
- 通过 OCC 页面手动取消
- 工作量大（137 个任务）
- 一次性操作

**方案 C：开发删除 API**
- 在 OCC 中添加删除端点
- 需要开发时间
- 长期解决方案

**建议采用方案 A**，理由：
1. 无需技术操作
2. 保留历史记录
3. 新系统自然覆盖

### 问题 2：中文名和拼音目录混存

**问题描述：**
```bash
ls runtime/inboxes/
# 输出：
drwxr-xr-x  jiangxin/     # ✅ 拼音
drwxr-xr-x  lingxi/       # ✅ 拼音
drwxr-xr-x  panshi/       # ✅ 拼音
drwxr-xr-x  siku/         # ✅ 拼音
drwxr-xr-x  tianshu/      # ✅ 拼音
drwxr-xr-x  tiance/       # ✅ 拼音
drwxr-xr-x  zhiju/        # ✅ 拼音
drwxr-xr-x  匠心/         # ⚠️ 中文（旧数据）
drwxr-xr-x  司库/         # ⚠️ 中文（旧数据）
drwxr-xr-x  天枢/         # ⚠️ 中文（旧数据）
drwxr-xr-x  执矩/         # ⚠️ 中文（旧数据）
```

**影响：**
- 新旧目录并存
- 可能导致消息分散
- 影响：中

**根本原因：**
- v3.2 修复前使用中文名目录
- v3.2 修复后使用拼音目录
- 旧目录未清理

**解决方案：**

**立即执行：**
```bash
# 1. 备份旧目录
mv runtime/inboxes/匠心 runtime/inboxes/jiangxin-old
mv runtime/inboxes/司库 runtime/inboxes/siku-old
mv runtime/inboxes/天枢 runtime/inboxes/tianshu-old
mv runtime/inboxes/执矩 runtime/inboxes/zhiju-old

# 2. 检查旧目录消息
ls runtime/inboxes/jiangxin-old/

# 3. 如有未读消息，迁移到新目录
cp runtime/inboxes/jiangxin-old/msg-*.json runtime/inboxes/jiangxin/

# 4. 删除旧目录
rm -rf runtime/inboxes/jiangxin-old
```
