# BCE 任务 1 分钟反馈机制

**版本：** v1.0  
**创建时间：** 2026-03-22 14:40  
**维护人：** 匠心 (CTO)  
**批准人：** 天枢 (CEO)

---

## 🎯 核心要求

**发现待办任务后，必须在 1 分钟内完成反馈！**

---

## 📋 反馈时限

| 场景 | 时限 | 超时处理 |
|------|------|---------|
| **查询到待办任务** | 1 分钟内 | 超时上报天枢 + 磊哥 |
| **打卡查询** | 每 5 分钟 | 超时上报天枢 + 磊哥 |
| **任务确认** | P0:30 分钟 | 超时通知创建人 |

---

## 🔄 工作流程

```
每 5 分钟 → 打卡脚本运行
    ↓
查询 BCE 任务
    ↓
发现待办任务 → 记录发现时间
    ↓
设置 1 分钟反馈截止时间
    ↓
保存待反馈任务到 pending_feedback.json
    ↓
监督脚本检查（每 5 分钟）
    ↓
发现超过 1 分钟未反馈 → 🚨 上报天枢 + 磊哥
```

### 2. 监督脚本 (bce-supervisor.py)

**检查逻辑：**

```python
# 读取待反馈文件
feedback_file = '~/.openclaw/agents/jiangxin/pending_feedback.json'

with open(feedback_file) as f:
    data = json.load(f)

deadline = data['feedback_deadline']
now = datetime.now().timestamp()

if now > deadline:
    # 超时！上报
    elapsed = now - data['found_timestamp']
    log("🚨 [匠心] 发现待办任务后已 " + str(int(elapsed)) + " 秒未反馈")
else:
    remaining = deadline - now
    log("⏳ [匠心] 待反馈任务进行中（剩余 " + str(int(remaining)) + " 秒）")
```

## ✅ 反馈方式

### 方式 1：BCE 系统 Web 界面

1. 访问：http://192.168.31.187:3000/bce-tasks.html
2. 登录个人账户
3. 找到待确认任务
4. 点击"确认"按钮
5. ✅ 反馈完成（pending_feedback.json 自动清除）

### 方式 2：飞书确认

1. 在飞书群收到任务通知
2. 点击"确认任务"按钮
3. ✅ 反馈完成


## 📊 监督机制

### 检查频率

**每 5 分钟检查一次：**

```
*/5 * * * * python3 /Users/ai/.openclaw/agents/jiangxin/scripts/bce-supervisor.py
```

### 检查内容

1. **打卡情况** - 每 5 分钟是否查询
2. **反馈情况** - 发现待办后 1 分钟内是否反馈

### 违规处理

| 违规类型 | 处理措施 |
|---------|---------|
| 超时 5 分钟未打卡 | 记录违规 + 上报天枢 + 磊哥 |
| 超时 1 分钟未反馈 | 记录违规 + 上报天枢 + 磊哥 |

## 🚨 异常处理

### 场景 1：忘记反馈

**现象：** 超过 1 分钟未反馈

**处理：**
1. 监督脚本检测到超时
2. 记录违规日志
3. 上报天枢 + 磊哥
4. 本人立即反馈

### 场景 2：已反馈但文件未清除

**现象：** pending_feedback.json 未自动清除

**处理：**
```bash
# 手动清除
rm ~/.openclaw/agents/jiangxin/pending_feedback.json
```

### 场景 3：BCE 系统故障

**现象：** 无法访问 BCE 系统，无法反馈

**处理：**
1. 记录故障时间
2. 使用备用方式反馈（飞书/Skill）
3. 故障期间时限顺延
