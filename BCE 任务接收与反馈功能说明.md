## 🎯 核心目标

### 设计理念

**培养主动意识，杜绝被动等待：**

1. ✅ **主动查询** - 每 5 分钟主动查询任务，不等待通知
2. ✅ **及时确认** - 查到任务立即确认，不拖延
3. ✅ **主动反馈** - 执行过程主动更新进度
4. ✅ **按时完成** - 严格按照优先级时限完成

### 关键原则

| 原则 | 说明 | 违规处理 |
|------|------|---------|
| **主动查询** | 每 5 分钟查询一次 | 超时上报天枢 + 磊哥 |
| **及时确认** | P0:30 分钟，P1:2 小时，P2:24 小时，P3:72 小时 | 超时通知创建人 |
| **1 分钟反馈** | **发现待办任务后 1 分钟内必须反馈** | **超时上报天枢 + 磊哥** |
| **主动反馈** | 执行过程主动更新进度 | 纳入绩效考核 |
| **按时完成** | 严格按照 deadline 完成 | 影响任务评级 |

## 📥 任务接收流程

### 流程图

```
任务创建 (天枢/司库/其他人)
    ↓
BCE 系统分配任务
    ↓
定时轮询检测 (每 5 分钟)
    ↓
成员查询到任务
    ↓
成员手动确认任务
    ↓
开始执行任务
```

### 任务创建

**创建人：** 天枢、司库、或任何有权限的成员

**创建方式：**
1. BCE 系统 Web 界面创建
2. 通过 bce-task-flow Skill 自然语言创建
3. API 调用创建

**必填信息：**
- 任务标题
- 任务描述
- 负责人 (assignee)
- 优先级 (P0/P1/P2/P3)
- 截止时间 (dueDate)

**示例：**
```
标题：🚀 北斗智投系统 - 财务知识库
描述：负责财务知识库搭建
负责人：司库
优先级：P0
截止时间：2026-03-22 17:00
```

## 🔄 定时轮询机制

### 核心机制

**不是"提醒"，是"强制检查"：**

```
┌──────────────────────────────────────────────────────────────┐
│                     每个人的 Agent                            │
│                                                              │
│  ┌────────────────────────────────────────────────────────┐ │
│  │  定时任务 (每 5 分钟)                                     │ │
│  │  */5 * * * * python3 bce-checkin.py                    │ │
│  │                        ↓                                │ │
│  │  1. 调用 BCE API 查询"我的待办任务"                       │ │
│  │  2. 记录打卡时间到 last_check_in.json                   │ │
│  │  3. 发现待确认任务 → 显示详情（不自动确认！）           │ │
│  │  4. 记录日志                                             │ │
│  └────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
                            ↓
┌──────────────────────────────────────────────────────────────┐
│                    监督脚本 (每 5 分钟)                        │
│                                                              │
│  ┌────────────────────────────────────────────────────────┐ │
│  │  bce-supervisor.py                                      │ │
│  │                        ↓                                │ │
│  │  1. 检查所有人的 last_check_in.json                     │ │
│  │  2. 计算距离上次打卡的时间                               │ │
│  │  3. 超过 5 分钟未打卡 → ❌ 上报天枢 + 磊哥               │ │
│  │  4. 记录监督日志                                         │ │
│  └────────────────────────────────────────────────────────┘ │
└──────────────────────────────────────────────────────────────┘
```

### 打卡脚本详解

**脚本位置：** `/Users/ai/.openclaw/agents/{agent}/scripts/bce-checkin.py`

**定时任务：**
```bash
*/5 * * * * python3 /Users/ai/.openclaw/agents/{agent}/scripts/bce-checkin.py
```

**执行逻辑：**
```python
1. 调用 BCE API
   GET /api/bce/tasks?assignee={名字}&status=pending

2. 解析返回数据
   - 任务列表
   - 任务详情（标题、优先级、状态、创建时间）

3. 筛选未确认任务
   - confirmedAt 为空的任务

4. 显示待办任务详情
   - 任务标题
   - 优先级
   - 状态
   - 已过时间
   - 确认时限
   - 剩余时间

5. 记录打卡时间
   - 写入 last_check_in.json
   - 证明自己查过了

6. 写入日志
   - 打卡日志
   - 待办任务详情
```

**打卡文件示例：**

### 监督脚本详解

**脚本位置：** `/Users/ai/.openclaw/agents/jiangxin/scripts/bce-supervisor.py`

**定时任务：**
```bash
*/5 * * * * python3 /Users/ai/.openclaw/agents/jiangxin/scripts/bce-supervisor.py
```

**执行逻辑：**
```python
1. 遍历所有成员的打卡记录
   USERS = ['匠心', '司库', '磐石', '执矩', '灵犀', '天策', '智源']

2. 检查每个人的 last_check_in.json
   - 读取最后打卡时间
   - 计算当前时间 - 最后打卡时间

3. 判断是否超时
   - 如果 > 5 分钟：❌ 违规
   - 如果 ≤ 5 分钟：✅ 正常

4. 违规处理
   - 记录违规日志
   - 上报天枢 + 磊哥
   - 通知违规成员

5. 写入监督日志
```

**监督逻辑：**
```python
CHECK_IN_INTERVAL = 5 * 60  # 5 分钟

for user in USERS:
    last_check_in = 读取打卡文件
    elapsed = 现在时间 - last_check_in
    
    if elapsed > 5 分钟:
        ❌ 上报：{user.name} 已 {elapsed} 分钟未查询任务
    else:
        ✅ 正常：{user.name} {elapsed} 分钟前已查询
```

## ✅ 任务确认流程

### 确认时限

| 优先级 | 确认时限 | 超时处理 |
|--------|---------|---------|
| **P0** | 30 分钟 | 超时 5 分钟第 2 次提醒，15 分钟通知创建人，30 分钟上报磊哥 |
| **P1** | 2 小时 | 超时 30 分钟第 2 次提醒，1 小时通知创建人，2 小时上报磊哥 |
| **P2** | 24 小时 | 超时 6 小时第 2 次提醒，12 小时通知创建人，24 小时上报磊哥 |
| **P3** | 72 小时 | 超时 12 小时第 2 次提醒，24 小时通知创建人，72 小时上报磊哥 |

### 确认方式

**方式 1：BCE 系统 Web 界面**

1. 访问 BCE 系统：http://192.168.31.187:3000/bce-tasks.html
2. 登录个人账户
3. 查看"我的任务"列表
4. 找到待确认任务
5. 点击"确认"按钮
6. 任务状态更新为"已确认"

**方式 2：bce-task-flow Skill**

```
用户：确认任务 [任务 ID 或任务标题]

Skill 响应：
✅ 任务已确认
- 确认人：匠心
- 确认时间：2026-03-22 14:30

```

**方式 3：API 调用**

```bash
curl -X POST http://localhost:3000/api/bce/tasks/{task_id}/confirm \
```

**执行过程中必须主动反馈：**

| 任务时长 | 反馈频率 | 反馈内容 |
|---------|---------|---------|
| < 1 小时 | 完成后反馈 | 完成结果 |
| 1-4 小时 | 每 1 小时反馈 | 进度百分比、遇到的问题 |
| 4-24 小时 | 每 2 小时反馈 | 进度百分比、已完成部分、下一步计划 |

4. 记录驳回次数

**驳回阈值：**
- 同一任务被驳回 **5 次以上** → 🚨 通知天枢 + 磊哥

## 🚨 异常情况处理

### 场景 1：忘记打卡

**现象：** 超过 5 分钟未查询任务

**处理：**
1. 监督脚本检测到超时
2. 记录违规日志
3. 上报天枢 + 磊哥
4. 本人立即补打卡

**预防措施：**
- 设置个人提醒
- 养成习惯
- 互相监督

### 场景 2：BCE 系统故障

**现象：** 无法访问 BCE 系统

**处理：**
1. 检查 BCE 服务状态
2. 重启 BCE 服务
3. 记录故障时间
4. 故障期间任务确认时限顺延

**应急方案：**
- 使用备用 API 地址
- 临时使用飞书任务
- 手动记录任务

### 场景 3：任务分配错误

**现象：** 任务分配给错误的人

**处理：**
1. 发现人立即通知创建人
2. 创建人修改任务负责人
3. 记录转交历史
4. 重新计算确认时限

### 场景 4：任务无法完成

**现象：** 遇到技术难题或其他原因无法完成

**处理：**
1. 立即在 BCE 系统标记"阻塞"
2. 详细说明原因
3. 通知创建人和审核人
4. 申请协助或调整任务

```bash
mkdir -p /Users/ai/.openclaw/agents/{agent_name}/scripts
```

```bash
vi /Users/ai/.openclaw/agents/{agent_name}/scripts/bce-checkin.py
```

**步骤 4：添加定时任务**
```bash
(crontab -l; echo "*/5 * * * * python3 /Users/ai/.openclaw/agents/{agent_name}/scripts/bce-checkin.py") | crontab -
```

**步骤 6：测试验证**
