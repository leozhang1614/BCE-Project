## 附录 B：Control Center 详细功能说明

### B.1 系统监控

**功能说明：**
- 实时监控系统健康状态
- CPU/内存使用率统计
- 服务运行状态检查
- 错误日志查看

**访问方式：** `http://192.168.31.187:4310` → 系统监控

**监控指标：**
| 指标 | 说明 | 正常范围 |
|------|------|---------|
| 服务状态 | BCE、Control Center 运行状态 | ✅ 运行中 |
| CPU 使用率 | 系统 CPU 占用 | < 80% |
| 内存使用 | 系统内存占用 | < 80% |
| 运行时间 | 服务连续运行时间 | - |
| 最后检查 | 最后健康检查时间 | < 1 分钟 |

### B.2 会话管理

**功能说明：**
- 管理 OpenClaw 会话状态
- 查看活跃会话列表
- 会话历史记录
- 会话终止/重启

**会话状态：**
| 状态 | 说明 | 操作 |
|------|------|------|
| Active | 活跃会话 | 可查看/终止 |
| Idle | 空闲会话 | 可查看/重启 |
| Terminated | 已终止 | 可查看历史 |

### B.3 任务展示（只读）

**功能说明：**
- 展示 BCE 同步的任务数据
- 30 秒自动刷新
- 支持任务搜索和筛选
- 只读模式，不可编辑

## 附录 E：故障排查清单

### E.1 BCE 服务故障

**检查步骤：**
1. 检查服务状态：`curl http://localhost:3000/health`
2. 检查数据文件：`ls -la runtime/bce-data.json`
3. 查看服务日志：`tail -f logs/app.log`
4. 重启服务：`npm start`

### E.2 飞书通知故障

**检查步骤：**
1. 检查.env 配置：`cat .env`
2. 检查 Token 获取：查看日志 `[飞书] Token 获取成功`
3. 检查@user_id：查看日志 `[飞书] 设置@用户`
4. 检查机器人权限：飞书开放平台 → 权限管理

### E.3 数据同步故障

**检查步骤：**
1. 检查同步脚本：`ps aux | grep sync-to-cc`
2. 手动运行同步：`node scripts/sync-to-cc.js`
3. 检查 CC 数据文件：`ls -la openclaw-control-center/runtime/tasks.json`
4. 查看同步日志
