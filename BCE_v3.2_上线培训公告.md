## 📚 系统概述

### 什么是 BCE？

**BCE（北斗协同引擎）** 是一款基于 OpenClaw 的智能任务管理和协同系统，帮助团队高效分配任务、跟踪进度、确保任务按时完成。

### 核心能力

| 能力 | 说明 | 例子 |
|------|------|------|
| **任务分配** | 把任务分配给具体的人 | 天枢分配任务给匠心 |
| **及时通知** | 任务变动及时通知相关人员 | 匠心收到 sessions_send 消息 |
| **进度跟踪** | 实时查看任务做到哪一步了 | 查看任务看板 |
| **任务流转** | 任务在不同人之间传递 | 匠心→司库→执矩（自动） |
| **完成确认** | 确保任务被确认和执行 | 匠心点击卡片按钮 |
| **确认反馈** | 上一节点收到确认通知 | 天枢收到"匠心已确认" |

### v3.2 核心改进

| 功能 | v3.0 | v3.2 | 提升 |
|------|------|------|------|
| **通知通道** | 单通道 | 三通道 | ✅ 更可靠 |
| **流转方式** | 手动调用 | 规则引擎 | ✅ 自动流转 |
| **消息匹配** | 关键词 | parent_id 精确匹配 | ✅ 可落地 |
| **确认反馈** | 无 | 通知上一节点 | ✅ 闭环 |
| **循环检测** | 无 | 自动阻止 | ✅ 防止死循环 |
| **数据持久化** | 部分 | 100% 端点 | ✅ 无丢失 |

## 🎯 使用要求

### 1. 全员必须使用

**适用范围：** 天枢、匠心、司库、执矩、磐石、灵犀、天策（7 人）

**强制要求：**
- ✅ 所有任务必须通过 BCE 系统创建和分配
- ✅ 任务状态变更必须通过系统执行
- ✅ 任务确认必须使用系统确认功能
- ✅ 禁止私下分配任务（特殊情况除外）

### 2. 任务确认时效

| 优先级 | 确认时限 | 超时处理 |
|--------|---------|---------|
| **P0** | 30 分钟内 | 系统自动提醒 |
| **P1** | 2 小时内 | 系统自动提醒 |
| **P2** | 24 小时内 | 系统自动提醒 |
| **P3** | 72 小时内 | 系统自动提醒 |

### 3. 任务流转规范

**标准流程：**
```
创建 → 分配 → 确认 → 执行 → 提交 → 验收 → 完成
```

**自动流转规则：**
- 技术方案完成 → 自动转交司库（财务审核）
- 财务审核完成 → 自动转交执矩（安全审核）
- 安全审核完成 → 自动转交天枢（CEO 验收）

### 4. 通知响应

**三通道通知：**
1. **主通道：** OpenClaw sessions_send（实时推送）
2. **备用通道：** 飞书卡片（手机端提醒）
3. **可靠通道：** 文件系统邮箱（离线拉取）

**响应要求：**
- 收到通知后**必须确认**（点击卡片按钮或回复"收到"）
- 如有异议，使用**驳回**功能并说明原因
- 禁止忽略通知

### 5. 数据规范

**任务创建规范：**
- 标题：清晰描述任务内容
- 优先级：P0/P1/P2/P3（必须指定）
- 负责人：明确指定（不能为空）
- 截止时间：如有要求，必须填写

**任务状态更新：**
- 开始执行：必须点击 [开始执行]
- 完成任务：必须点击 [完成] 触发自动流转
- 遇到问题：必须标记为 [阻塞] 并说明原因
